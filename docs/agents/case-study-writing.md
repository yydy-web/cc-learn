## 场景概述

客户成功团队发来一堆素材："A 公司用了我们产品后效果不错，帮忙包装一下做案例。"素材包里有：客户行业背景、使用时长、3 个使用数据、销售和客户通的 10 分钟电话摘要、客户邮件里的一句"挺好用的，效率高了不少"。

包装一个客户案例不只是"写篇文章"。你需要产出：

- **官网案例摘要**：120 字以内，让人一眼扫完就知道"什么客户、做了什么、效果如何"
- **结构化案例故事**：背景 → 挑战 → 方案 → 结果，每段有数据和引文
- **社交媒体版本**：LinkedIn 的专业语气 + 微信公众号的"有料"语气——两份完全不同的文案
- **销售话术**：基于这个案例提炼 3 句销售跟客户聊天时能直接用的话

传统做法是市场部排期、文案写初稿、PM Review 数据、设计师配图——至少 2-3 个工作日的协作。Agent 能把这个流程压缩到 5 分钟：你扔给它原始素材和输出规格，它一次性产出全部版本。

## 为什么用 Agent 做案例包装

| 对比维度   | 人工包装                         | Agent 包装                                       |
| ---------- | -------------------------------- | ------------------------------------------------ |
| 产出速度   | 2-3 天（撰稿 + 审核 + 修改）     | 5 分钟（生成多版本初稿）                         |
| 多版本适配 | 通常只写一个版本，改编靠人工     | 同时输出官网版 + LinkedIn 版 + 微信版 + 销售话术 |
| 数据精度   | 容易夸大或模糊数据（"大幅提升"） | 严格基于你给的数据，不添油加醋                   |
| 一致性     | 不同案例可能风格不一             | 同一个人设 prompt → 统一风格                     |
| 人力成本   | 市场和 PM 各投入 2-4 小时        | 0 人力成本（素材准备 + 审稿 15 分钟）            |

核心不是"AI 写出了比文案更好的文案"，而是**AI 把草稿写好了，人的工作从"写"变成了"审"**——审稿比写稿快 5 倍。

## 前置准备

准备一个素材包，建议包含：

- 客户名称和行业
- 使用产品的时长
- 使用前后对比数据（至少 2-3 个可量化指标）
- 客户原话摘录（哪怕只是邮件里的几句话）
- 使用产品的关键场景（哪个人、做什么事、原来怎么做、现在怎么做）

:::tip
素材包里最重要的是"数据"和"原话"。数据是案例说服力的来源，原话是案例人情味的来源。两者缺一个，案例要么太干要么太虚。
:::

## 完整交互过程

### Step 1：把素材和输出规格给 Agent

```text
你是资深 SaaS 内容营销经理。请根据以下客户素材，生成客户案例内容包。

**客户素材**
- 客户名称：卫通科技（化名）
- 行业：物流 SaaS
- 规模：200 人，研发团队 40 人
- 使用产品：我们的数据导出工具
- 使用时长：4 个月
- 使用前：运营同事每周手动从数据库导 3 次数据做周报，每次 2-3 小时
- 使用后：配置了定时导出 + Webhook → 飞书群自动推送，周报时间从 3 小时降到 10 分钟
- 客户原话摘录：
  - "以前周五下午就是导出数据、做报表，现在可以早点回家"
  - "数据一旦出错就重新导，最烦的是导到一半断网"
  - "你们的 webhook 功能好——导完后飞书自动发消息，不用我们蹲在那等"

**输出规格**
1. 120 字摘要（用于官网首页案例轮播）
2. 结构化案例故事（背景/挑战/方案/结果，每段 ≤ 100 字）
3. 3 个可引用的数据亮点
4. LinkedIn 版文案（专业语气，300 字以内 + 3 个 hashtag）
5. 微信公众号版文案（口语化，有画面感，500 字以内）
6. 销售话术：3 句可以直接用的"故事化价值主张"
```

### 2. 结构化案例故事

**背景**
卫通科技是一家 200 人的物流 SaaS 企业，运营团队每周需要从数据库
导出订单数据，生成周报给管理层和客户。这条路走了两年——手工、重复、
容易出错。

**挑战**
- 每周导出 3 次，每次 2-3 小时，占运营 15% 的工作时间
- 导出过程中断网就得重来
- 数据出错时无法追溯是导出环节的问题还是数据本身的问题

**方案**
- 配置了 3 个定时导出任务（订单数据 / 运单数据 / 客户报表）
- 导出完成后 Webhook 自动推送到飞书群
- 运营同事只需打开群聊下载文件，不再需要登录数据库操作

**结果**
- 周报准备时间：3 小时 → 10 分钟（效率提升 94%）
- 运营每周节省约 9 小时，相当于多出一个人一天的工作量
- 零手动导出错误，数据质量问题投诉减少

### Step 3：团队审核 + 调整敏感信息

Agent 产出的初稿经过两层审核：

1. **数据审核**（PM）：确认所有数字准确、有来源支撑、不存在夸大
2. **品牌审核**（市场）：确认语气、措辞与品牌调性一致、客户名称和行业是否符合隐私协议（化名或真名需客户授权）

修改通常是微调——把某个数据从"4 个月"改成"2026 年 Q1 以来"、把微信文案的标题改得更吸引人。整体骨架不需要动，因为 Agent 只基于你给的数据，不会凭空捏造。

## 要点总结

1. **素材质量 = 案例质量**。Agent 不会虚构数据，你给多少它就写多少。素材包中数据和原话是两个核心支柱——缺一个，案例就是瘸腿的。
2. **多版本复用是核心价值**。官网、LinkedIn、公众号、销售话术是四套完全不同语气和结构的文案，Agent 一次产出全部——人工写至少需要切换四次思维模式。
3. **销售话术是最高 ROI 的产出物**。一个好的客户案例能让销售团队的话术库多出 3 条可复用的"故事化价值主张"，每条都可能帮助推进一个 deal。
4. **审稿比写稿重要**。Agent 产出的是"草稿"，不是"终稿"。你需要检查的是：数据是否准确、语气是否符合品牌、有没有过度承诺。
5. **案例库积累产生复利**。做了 5 个案例后，Agent 的风格会越来越接近你的品牌调性——把之前的案例作为 reference 输入，新案例产出的一致性会持续提升。

## 变体与延伸

- **行业解决方案页**：不止是一个客户案例，而是同类客户的共性挑战、方案和结果聚合，用于官网解决方案页
- **产品更新公告**：把 Release Notes（功能清单 + 技术细节）变成用户能看懂的"这个更新对你意味着什么"
- **投资者/媒体 FAQ**：把公司数据、产品进展、市场定位交给 Agent，生成投资者沟通 FAQ 和媒体通稿 FAQ
- **竞品对比页**：基于竞品调研矩阵，生成"我们 vs X"的官网对比页文案
