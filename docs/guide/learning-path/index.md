# 学习路线

每个人的背景和目标不同，学 Claude Code 的路径也不该一样。选一个最接近你的角色，走那条最适合的路。

## 选择你的角色

| 角色 | 一句话描述 | 预计时间 | 入口 |
|------|-----------|----------|------|
| 🎯 [不懂编程](/guide/learning-path/non-programmer) | 没用过命令行？没写过代码？从这里开始 | ~2 小时 | [开始 →](/guide/learning-path/non-programmer) |
| 📋 [产品经理](/guide/learning-path/product) | 写 PRD、做原型、管需求、出汇报 | ~3 小时 | [开始 →](/guide/learning-path/product) |
| 🎨 [前端开发](/guide/learning-path/frontend) | 写页面、调样式、接 API、搞设计还原 | ~5 小时 | [开始 →](/guide/learning-path/frontend) |
| ⚙️ [后端开发](/guide/learning-path/backend) | 写接口、管数据库、做架构、搭 CI/CD | ~5 小时 | [开始 →](/guide/learning-path/backend) |
| 🧪 [测试](/guide/learning-path/qa) | 写用例、做自动化、查 Bug、出报告 | ~4 小时 | [开始 →](/guide/learning-path/qa) |

## 不知道选哪个？

先走 [不懂编程](/guide/learning-path/non-programmer) 路线。这是所有人共同的起点——安装、第一次对话、基础操作。走完大约 2 小时后，再根据你的实际工作需要跳到对应角色路线继续。

## 已熟悉基础？

直接跳对应的专项路线：
- 写需求的 → [产品经理](/guide/learning-path/product) 阶段三开始
- 写页面的 → [前端开发](/guide/learning-path/frontend) 阶段四开始
- 写接口的 → [后端开发](/guide/learning-path/backend) 阶段四开始
- 做测试的 → [测试](/guide/learning-path/qa) 阶段三开始
