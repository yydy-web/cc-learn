# 全局菜单审计与文档全量更新 — 设计规格

## 目标

对 CC Learn 文档站（128 页）进行全局菜单一致性审计和内容全量更新，按导航分区迭代执行，最终统一文档质量和导航结构。

## 审查标准（11 项）

每页审查时对照：

1. **description frontmatter** — 存在，50-160 字，准确描述页面内容
2. **内容时效** — 描述的功能/命令在当前最新版 Claude Code 仍然有效
3. **归属正确** — 页面在合理的章节/目录下，无放错位置
4. **导航一致** — 所在目录的 `_meta.json` 包含该页，无孤儿页、无死链
5. **写作质量** — 无错别字、语法错误、逻辑断裂
6. **交叉引用** — 相关内容之间有链接串联，读者可以自然发现关联页面
7. **代码块语法** — ` ``` ` 的语言标识正确，可执行片段无语法错误
8. **功能串联** — 页面内容涉及的上下游功能/工具之间有明确的关联指引
9. **上手星级** — 针对产品/测试/开发三种角色各给出推荐度（★）和上手难度（★）
10. **学习产出** — 一句话描述使用后的收获
11. **AI 能力提升** — 提升 Claude Code 的哪方面能力

## 新增：页面导航卡片

每页 body 开头插入 `:::info` 容器，包含：

- **推荐度**（★）：该角色看这页的实用程度。5★ = 必须看，1★ = 可选
- **上手难度**（★）：该角色理解/操作这页内容的门槛
- **学习产出**：一句话描述使用后的收获
- **AI 能力提升**：从固定标签集中选择

### 固定标签集

`上下文管理` `代码生成` `自动化工作流` `代码审查` `测试生成` `项目协作` `技能扩展` `调试诊断` `性能优化` `跨文件重构` `设计→代码` `多智能体` `规格驱动`

### 卡片模板

```markdown
:::info {title="📊 页面导航"}
**适用角色与上手难度**

| 角色 | 推荐度 | 上手难度 |
|------|--------|----------|
| 🛠️ 开发 | ★★★★★ | ★★☆☆☆ |
| 🧪 测试 | ★★★☆☆ | ★★★☆☆ |
| 📦 产品 | ★★☆☆☆ | ★★★★☆ |

**🎯 学习产出：** 掌握 xxx，能独立 xxx

**🚀 AI 能力提升：** xxx（上下文管理 / 代码生成 / ...）
:::
```

## 菜单审计方法

**检查范围：** 所有 `_nav.json` 和 `_meta.json` 文件。

**检查项：**

| 检查项 | 方法 |
|--------|------|
| 孤儿文件 | `find` 全量 .md 文件 vs `_meta.json` 引用集合 |
| 死链 | `_meta.json` 中每个 key 对应 .md 文件存在 |
| 归属合理性 | 人工判断页面是否在正确的章节/目录下 |
| 排序合理性 | 页面顺序是否符合学习路径（基础→进阶→高级） |
| 标签一致性 | section-header 标签与内容实际是否匹配 |

**产出：** 每阶段生成审计报告，列出发现 + 修复建议，修复后删除。

## 执行策略

### 分批迭代（4 阶段，每阶段 1 个 PR）

| 阶段 | 分区 | 预估页数 | 产出 |
|------|------|----------|------|
| 1 | 教程 (guide/) | ~50 | 菜单修复 + 全页审查 + 卡片 |
| 2 | 技巧 (tips/) | ~38 | 菜单修复 + 全页审查 + 卡片 |
| 3 | 技能 (skills/) | ~16 | 菜单修复 + 全页审查 + 卡片 |
| 4 | 命令 (commands/) | ~12 | 菜单修复 + 全页审查 + 卡片 |

### 每阶段流程

```
菜单审计 → 逐页审查 → 批量修复 → npm run build 验证
```

### 逐页审查流程

1. 读文件
2. 对照 11 项标准逐项检查
3. 修复发现的问题
4. 插入导航卡片
5. 更新所在 `_meta.json`（如有新增/删除）

## 验证

- 每阶段完成后运行 `npm run build`，确认零错误
- 构建产物检查：所有页面可访问、无 404、导航卡片正确渲染
- 每阶段提交前运行 `npm run check`（lint + format）

## 边界

- 不包含 `docs/superpowers/specs/` 和 `docs/compose/` 目录下的设计文档/计划文档
- 不修改 `rspress.config.ts`、主题文件、CSS
- 不新增或删除页面，除非审计发现确实需要的调整
- 外部链接页面（如 Claude Code 官方文档）不在审查范围内
